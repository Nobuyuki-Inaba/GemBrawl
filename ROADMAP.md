## Phase 3 — ネットワーク4人対戦

同一WiFi（LAN）内のプレイヤー同士が前提。外部サービス完全不要。

**接続フロー（スタートポロジー）:**
```
① ホスト(P1)が「ルーム作成」→ ルームコードが表示され自動クリップボードコピー
② P1がLINE/チャットでコードを参加者に送る
③ 参加者(P2〜P4)がコードを貼り付け → 応答コードが自動コピーされる
④ 参加者が応答コードをP1に送り返す
⑤ P1が応答コードを貼り付け → 接続確立
（LAN内なのでSTUN/TURNは不要。WebRTCがローカルIPを直接使う）
```

**UI設計方針（ライトユーザー向け）:**
- コードは「コピー」ボタン1つでクリップボードへ自動コピー
- 画面に日本語で手順を番号付きで表示
- 入力ボックスは大きく、貼り付けやすく

- [ ] WebRTC RTCDataChannel の手動シグナリングUI（コピペ方式）

- [ ] LockstepCoordinator（3フレーム入力遅延）

## 技術的リスクと対策

| リスク | 対策 |
|---|---|
| ブラウザ間の浮動小数点差異 | Phase 3まではChrome限定、または固定小数点演算 |
| 遅延プレイヤーによる全員止まり | 100ms超で前フレーム入力を予測、5連続失敗でドロップ |
| 別のWiFiから接続したい場合 | ZeroTierなどの無料VPNで同一LAN環境を作れば対応可 |
